GitHub MCP Server

Cómo usar este repositorio como contexto MCP

GitHub MCP Server no convierte este repositorio en una API jurídica pública por sí solo. Lo que hace es permitir que un cliente compatible con MCP acceda al repositorio de GitHub y utilice sus archivos como contexto para responder preguntas.

Para decisiones con efecto jurídico o académico relevante, verifique siempre la norma en su fuente oficial y la vigencia efectiva en BOE, BOC o portal institucional competente.

Usos recomendados

  • Buscar normativa en el repositorio.
  • Relacionar currículo, etapa y norma.
  • Pedir que la IA abra primero llms.txt.
  • Pedir revisión previa de 06_indices.

Casos de uso

Buscar en el repositorio

Pida al cliente MCP que abra el repositorio ateeducacion/normativa_educativa, lea llms.txt y localice archivos relevantes antes de responder.

Preguntar por una norma

Úselo para preguntas sobre evaluación, promoción, currículo, etapas o relaciones entre normas estatales y autonómicas, siempre con cita jurídica oficial.

Relacionar currículo, etapa y norma

Solicite que inspeccione 06_indices/curriculos.yaml, 06_indices/normativa.yaml y 06_indices/relaciones.yaml antes de sintetizar.

Trazabilidad sin exponer IDs

Los IDs internos sirven para localizar ficheros, pero la respuesta final debe citar “Ley Orgánica…”, “Real Decreto…”, “Decreto…” o “Orden…” según corresponda.

Configuración de ejemplo

VS Code / Copilot compatible

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@github/github-mcp-server"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "YOUR_TOKEN_HERE"
      }
    }
  }
}

Cliente MCP local genérico

{
  "servers": [
    {
      "name": "github",
      "transport": "stdio",
      "command": "github-mcp-server",
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "YOUR_TOKEN_HERE"
      }
    }
  ]
}

Servidor MCP con Docker

docker run --rm -i \
  -e GITHUB_PERSONAL_ACCESS_TOKEN=YOUR_TOKEN_HERE \
  ghcr.io/github/github-mcp-server:latest

Seguridad mínima recomendada

  • Use el token de GitHub con el menor privilegio posible.
  • Para hacer preguntas, el acceso de solo lectura suele ser suficiente.
  • No suba nunca un archivo .env con secretos al repositorio.
  • Este repositorio ignora .env; mantenga también fuera de control cualquier variante sensible.
  • Prefiera acceso a repositorios públicos cuando sea suficiente.
  • Verifique permisos antes de habilitar herramientas que puedan escribir issues, pull requests o contenido.

Prompts recomendados

“Open the repository ateeducacion/normativa_educativa. Read llms.txt first. Then answer my question using the repository files as context. When citing regulations, use the official legal citation format used in Spain, not internal repository IDs.”

“Search the repository ateeducacion/normativa_educativa for current regulations about assessment and promotion in Educación Secundaria Obligatoria in Canarias. Use official legal titles, articles, and publication references when available.”

“Inspect the YAML indexes in 06_indices before answering. Use internal IDs only to locate files, not as the public citation format.”

El repositorio aporta contexto documental; no reemplaza las fuentes oficiales ni crea una API jurídica pública por sí mismo.
