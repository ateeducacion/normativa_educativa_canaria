GitHub MCP Server

Cómo usar este repositorio como contexto MCP

GitHub MCP Server permite que un cliente compatible con MCP (Model Context Protocol) acceda al repositorio en GitHub y use sus archivos como contexto para responder preguntas. No convierte por sí mismo este repositorio en una API jurídica pública.

Para decisiones con efecto jurídico o académico relevante, verifique siempre la norma en su fuente oficial y la vigencia efectiva en BOE, BOC o portal institucional competente.

Usos recomendados

  • Buscar normativa en el repositorio.
  • Relacionar currículo, etapa y norma.
  • Pedir que la IA abra primero llms.txt.
  • Pedir revisión previa de 06_indices.

Casos de uso

Buscar en el repositorio

Pida al cliente MCP que abra el repositorio ateeducacion/normativa_educativa_canaria, lea llms.txt y localice archivos relevantes antes de responder.

Preguntar por una norma

Úselo para preguntas sobre evaluación, promoción, currículo, etapas o relaciones entre normas estatales y autonómicas, siempre con cita jurídica oficial.

Relacionar currículo, etapa y norma

Solicite que inspeccione 06_indices/curriculos.yaml, 06_indices/normativa.yaml y 06_indices/relaciones.yaml antes de sintetizar.

Trazabilidad sin exponer IDs

Los IDs internos sirven para localizar ficheros, pero la respuesta final debe citar “Ley Orgánica…”, “Real Decreto…”, “Decreto…” o “Orden…” según corresponda.

Cómo conectar GitHub MCP Server

GitHub mantiene el servidor oficial en github/github-mcp-server. Hay dos vías habituales hoy: el servidor remoto hospedado por GitHub y el servidor local en Docker. La vía remota es la recomendada cuando el cliente la admite, porque no requiere instalación.

El paquete antiguo npx @github/github-mcp-server y @modelcontextprotocol/server-github están deprecados desde abril de 2025. No se recomiendan.

Vía 1 — Servidor remoto hospedado (recomendado)

Endpoint público: https://api.githubcopilot.com/mcp/. Para un repositorio de consulta como este, conviene usar el modo de solo lectura: https://api.githubcopilot.com/mcp/x/repos/readonly.

VS Code 1.101 o superior

Cree .vscode/mcp.json en su espacio de trabajo. VS Code abrirá el flujo OAuth la primera vez; no necesita PAT si tiene Copilot habilitado.

{
  "servers": {
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp/x/repos/readonly"
    }
  }
}

Claude Code

Comando único en terminal con la versión actual del CLI:

claude mcp add-json github '{
  "type": "http",
  "url": "https://api.githubcopilot.com/mcp/x/repos/readonly",
  "headers": { "Authorization": "Bearer YOUR_GITHUB_PAT" }
}'

Cursor

En ~/.cursor/mcp.json o .cursor/mcp.json:

{
  "mcpServers": {
    "github": {
      "url": "https://api.githubcopilot.com/mcp/x/repos/readonly",
      "headers": {
        "Authorization": "Bearer YOUR_GITHUB_PAT"
      }
    }
  }
}

Claude Desktop

Claude Desktop no admite todavía OAuth contra el servidor remoto. Use la vía local con Docker.

Vía 2 — Servidor local con Docker

Imagen oficial: ghcr.io/github/github-mcp-server. Para Claude Desktop, edite ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) o el equivalente en otros sistemas:

{
  "mcpServers": {
    "github": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-e", "GITHUB_PERSONAL_ACCESS_TOKEN",
        "ghcr.io/github/github-mcp-server"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "TU_TOKEN_AQUI"
      }
    }
  }
}

Para forzar solo lectura sin pasar por la URL /readonly, añada GITHUB_READ_ONLY=1 en args y env.

Cómo se indica el repositorio

GitHub MCP Server no tiene un parámetro de configuración para "anclar" el servidor a un repositorio concreto. La selección del repo es siempre parte del prompt al modelo, usando las herramientas del toolset repos (get_file_contents, search_code, etc.). Por ejemplo:

Abre el repositorio ateeducacion/normativa_educativa_canaria. Lee primero llms.txt y después responde usando los archivos del repositorio como contexto.

Seguridad mínima recomendada

  • Use el token de GitHub con el menor privilegio posible.
  • Para hacer preguntas, el acceso de solo lectura suele ser suficiente. Use el endpoint /readonly o GITHUB_READ_ONLY=1.
  • No suba nunca un archivo .env con secretos al repositorio.
  • Este repositorio ignora .env; mantenga también fuera de control cualquier variante sensible.
  • Prefiera acceso a repositorios públicos cuando sea suficiente.
  • Verifique permisos antes de habilitar herramientas que puedan escribir issues, pull requests o contenido.

Prompts recomendados

“Abre el repositorio ateeducacion/normativa_educativa_canaria. Lee primero llms.txt. Después responde usando los archivos del repositorio como contexto. Al citar normas, usa la cita jurídica oficial habitual en España y no los IDs internos del repositorio.”

“Busca en el repositorio ateeducacion/normativa_educativa_canaria la normativa vigente sobre evaluación y promoción en Educación Secundaria Obligatoria en Canarias. Usa títulos oficiales, artículos y referencias de publicación cuando estén disponibles.”

“Inspecciona los índices YAML de 06_indices antes de responder. Usa los IDs internos solo para localizar archivos, no como formato público de cita.”

El repositorio aporta contexto documental; no reemplaza las fuentes oficiales ni crea una API jurídica pública por sí mismo.
