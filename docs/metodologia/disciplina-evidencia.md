# Disciplina de evidencia

Este repositorio aplica una disciplina de evidencia estricta para mantener un corpus IA-friendly, fiel y trazable.

## Principios

- **Sencillez primero.**
- **IA-friendly.**
- **Markdown para contenido narrativo.**
- **YAML para metadatos estructurados.**
- **Fuente oficial obligatoria.**
- **Fecha de consulta obligatoria.**
- **Fecha de análisis obligatoria cuando haya interpretación o resumen.**
- **Estado de vigencia obligatorio.**
- **Relaciones normativas explícitas.**
- **No copiar normas completas si basta con resumen, cita y enlace oficial.**
- **No mezclar normativa con recursos no normativos.**
- **No borrar contenido histórico: marcar como derogado, obsoleto o sustituido.**
- **Usar IDs estables y correlativos.**
- **Todo resumen debe indicar que no sustituye la fuente oficial.**
- **Todo índice debe ser legible por humanos y máquinas.**

## Regla operativa

No se afirma nada sobre una norma o currículo sin referencia a la fuente oficial, localización relevante y fecha de consulta. Cuando la evidencia es insuficiente se usa `[PENDIENTE]`, `[HIPÓTESIS]` o `[INTERPRETACIÓN]` según corresponda.

## Marcadores obligatorios

- `[HIPÓTESIS]`: afirmación razonable pendiente de comprobar.
- `[INTERPRETACIÓN]`: lectura editorial propia sobre un texto normativo.
- `[PENDIENTE]`: dato pendiente de verificar.
- `[OBSOLETO]`: contenido conservado solo por trazabilidad.
